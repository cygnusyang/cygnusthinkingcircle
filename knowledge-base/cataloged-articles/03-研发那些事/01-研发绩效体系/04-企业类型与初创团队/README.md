# 04-企业类型与初创团队

这个模块讨论不同价值创造方式下，研发绩效体系如何调整。

核心判断：

**不要照搬大厂绩效体系。先判断当前业务状态，再设计绩效逻辑。**

当前文章：

| 文件 | 说明 |
|------|------|
| 05-初创团队如何设计研发绩效.md | 讨论确定性运营、探索性创新、双元混合，以及初创团队实践 |

## 适合展开的主题

这个模块用“价值创造方式 -> 企业参照 -> 初创阶段 -> 业务形态 -> 技术形态”来拆。分类对象不是企业名称，而是研发团队所处的业务状态，因此更接近 MECE。

| 分类 | 可写主题 | 说明 |
|------|----------|------|
| 价值创造方式 | `确定性运营探索性创新双元混合.md` | 解释三种业务状态如何影响研发绩效 |
| 企业参照 | `大疆华为腾讯阿里的研发绩效参照意义.md` | 只讨论研发团队绩效，不泛化到全公司 |
| 初创阶段 | `初创团队不同阶段如何设计绩效.md` | 按 0->1、1->10、1->100 设计轻量绩效 |
| 业务形态 | `B2B和ToC研发绩效有什么不同.md` | 比较客户成功、用户反馈、交付周期和产品迭代差异 |
| 技术形态 | `硬件团队和软件团队绩效有什么不同.md` | 比较硬件、软件、AI、平台研发的评价差异 |
| 组织规模 | `从小团队到大团队绩效如何演进.md` | 讨论从创始人判断到组织机制的过渡 |

## 不放在本模块的内容

1. KPI 具体指标库放到 `01-KPI与研发绩效`；
2. 创新探索方法放到 `02-创新探索`；
3. 工程交付方法放到 `03-工程交付`；
4. 价值观评价方法放到 `06-组织与价值观`。
