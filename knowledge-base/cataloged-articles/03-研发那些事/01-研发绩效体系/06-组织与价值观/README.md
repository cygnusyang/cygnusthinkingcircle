# 06-组织与价值观

这个模块讨论价值观、行为质量和组织能力如何进入研发绩效判断。

核心判断：

**KPI 解决“看什么”，价值观解决“怎么做才算对”。**

当前文章：

| 文件 | 说明 |
|------|------|
| 09-研发绩效与价值观.md | 讨论价值观在研发绩效体系中的位置和风险 |

## 适合展开的主题

这个模块用“价值观 -> 行为质量 -> 奖惩 -> 管理者判断 -> 组织能力”来拆。它作为全系列最后一层，负责回答指标和方法无法完全覆盖的判断问题。

| 分类 | 可写主题 | 说明 |
|------|----------|------|
| 价值观 | `价值观如何转成可观察行为.md` | 把求真、客户价值、责任感、长期主义、协作转成行为标准 |
| 行为质量 | `低绩效不是配额.md` | 区分低绩效配额和低质量行为诊断 |
| 奖惩 | `奖惩机制如何避免伤害创新.md` | 讨论如何保护高质量探索，同时惩罚低质量行为 |
| 管理者判断 | `研发管理者如何做绩效判断.md` | 讨论指标之外的事实判断、校准和沟通 |
| 组织能力 | `绩效体系如何塑造组织能力.md` | 讨论绩效如何影响协作、沉淀、人才成长和长期竞争力 |

## 不放在本模块的内容

1. KPI 设计放到 `01-KPI与研发绩效`；
2. 创新项目评价放到 `02-创新探索`；
3. 工程质量和技术债放到 `03-工程交付`；
4. 模板放到 `05-框架与模板`。
