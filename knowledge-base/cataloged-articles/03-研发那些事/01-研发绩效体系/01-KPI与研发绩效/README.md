# 01-KPI与研发绩效

这个模块讨论 KPI 的边界。

核心判断：

**KPI 是观测工具，不是研发绩效本身。**

当前文章：

| 文件 | 说明 |
|------|------|
| 02-KPI与研发绩效关系.md | 解释 KPI 为什么不能直接替代研发绩效判断 |

## 适合展开的主题

这个模块用“定位 -> 误区 -> 设计 -> 使用 -> 校准”来拆。这样可以避免把 KPI 讨论和绩效方法论混在一起：本模块只讨论 KPI 作为指标工具的边界和用法。

| 分类 | 可写主题 | 说明 |
|------|----------|------|
| 定位 | `KPI在研发绩效体系中的位置.md` | 说明 KPI 是观测工具，不是绩效本身 |
| 误区 | `研发KPI常见误区.md` | 讨论完成率、缺陷数、工时、代码行数等常见误用 |
| 设计 | `不同阶段如何选择KPI.md` | 按 0->1、1->10、1->100 选择不同强度的 KPI |
| 设计 | `研发指标库.md` | 汇总交付、质量、效率、稳定性、沉淀等指标 |
| 使用 | `研发指标如何防止被博弈.md` | 讨论单一指标、强绑定奖惩带来的行为扭曲 |
| 校准 | `KPI什么时候应该调整.md` | 讨论目标、阶段、组织能力变化后的指标校准 |

## 不放在本模块的内容

1. 研发绩效的底层定义放到 `00-总纲`；
2. 探索项目怎么做放到 `02-创新探索`；
3. 交付项目怎么做放到 `03-工程交付`；
4. 奖惩和价值观边界放到 `06-组织与价值观`。
