# 05-框架与模板

这个模块把前面的讨论压缩成可复用的方法、框架和模板。

当前文章：

| 文件 | 说明 |
|------|------|
| 06-研发绩效方法论.md | 阶段判断、绩效结构、证据、KPI、奖惩、收敛机制 |
| 07-研发绩效管理框架一页纸.md | 一页纸框架、表格和评审模板 |
| 08-研发绩效阶段性总结.md | 对 KPI、研发绩效逻辑和方法论做阶段性收束 |

## 适合展开的主题

这个模块用“方法论 -> 框架图 -> 评审模板 -> 复盘模板 -> 指标库”来拆。它不再展开理论，而是把前面模块转成可复用工具。

| 分类 | 可写主题 | 说明 |
|------|----------|------|
| 方法论 | `研发绩效方法论.md` | 系统说明阶段判断、绩效结构、证据、KPI、奖惩、收敛机制 |
| 框架图 | `研发绩效管理框架一页纸.md` | 用一页纸压缩整体体系 |
| 评审模板 | `探索项目评审模板.md` | 用于 0->1 探索项目评审 |
| 评审模板 | `工程交付评审模板.md` | 用于交付、质量、技术债、稳定性评审 |
| 复盘模板 | `季度绩效复盘模板.md` | 用于团队季度目标、贡献、风险和沉淀复盘 |
| 指标库 | `研发指标库.md` | 汇总可选 KPI，但不替代绩效判断 |

## 不放在本模块的内容

1. 新理论和新概念优先放到对应主题模块；
2. 企业案例放到 `04-企业类型与初创团队`；
3. 价值观和行为边界放到 `06-组织与价值观`。
