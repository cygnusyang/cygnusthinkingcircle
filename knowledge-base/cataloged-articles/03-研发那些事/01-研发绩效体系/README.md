# 研发绩效体系

这个系列讨论的不是单纯的绩效考核，而是：

**研发组织如何在不确定性中创造价值、完成交付，并建立长期能力。**

研发绩效不能只从 KPI 开始。它要同时回答几个问题：

1. 研发到底是什么；
2. KPI 和研发绩效是什么关系；
3. 创新探索怎么评价；
4. 工程交付怎么评价；
5. 价值观如何进入绩效判断；
6. 不同业务状态和初创团队应该怎么设计绩效体系。

## 系列判断

研发绩效体系应该同时具备三件事：

1. **目标清楚**：知道要解决什么问题；
2. **方法有效**：知道如何探索、验证、收敛和交付；
3. **价值观明确**：知道什么行为值得鼓励，什么行为不能被奖励。

KPI 是其中的一部分，但不是全部。
